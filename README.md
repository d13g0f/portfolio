# About Me

I am a self-taught developer who started learning programming through game server development.

What originally began as a hobby experimenting with private MMORPG servers gradually evolved into a long-term learning journey. Over time, programming became a daily routine that I genuinely enjoy.

My first projects focused on modifying multiplayer game servers. Starting with MMORPG infrastructure exposed me early to complex systems such as:

- multiplayer gameplay logic
- game economy design
- exploit mitigation and server security
- gameplay scripting systems
- client/server tooling

Although MMORPG systems are often considered a difficult starting point due to their complexity, working with them helped me understand large interconnected systems from the beginning.

Most of my learning has come from building real projects, experimenting with gameplay systems, refactoring large codebases, and iterating over time.

Programming for me became not only a technical skill but also a mindset centered around patience, persistence, and continuous improvement.

# Projects

## [Tales of Pirates Server](https://github.com/d13g0f/portfolio/blob/main/Projects/Tales-of-Pirates-Server-Gameplay-Sys.md)
Gameplay Systems Refactoring & MMO Design

A heavily customized gameplay scripting layer for a Tales of Pirates MMORPG server built primarily in Lua.

The project focuses on refactoring and restructuring existing gameplay systems to make them clearer, scalable, and easier to maintain.

Main areas explored:

- combat and progression balancing
- modular Lua scripting architecture
- server event systems
- game economy design
- exploit research and mitigation

Development lasted approximately **three years**, during which nearly every gameplay system of the original game was analyzed, tested, and rebalanced.

## [TopModelPort / FBX Exporter](https://github.com/d13g0f/TopModelPort)
Legacy Asset Conversion Tool

A C++ desktop utility designed to convert proprietary game model formats into **FBX**.

The exporter reconstructs:

- skeleton hierarchies
- animation curves
- skinned meshes
- materials and textures

This allows legacy game assets to be reused in modern tools such as **Blender and Unity**.

# Development Philosophy

One of the most important lessons learned from working on these systems is that **there are no magic solutions in programming**.

Large systems improve through:

- patience
- iteration
- refactoring
- testing
- persistence

Many of the projects in this portfolio evolved through long periods of experimentation and redesign.
